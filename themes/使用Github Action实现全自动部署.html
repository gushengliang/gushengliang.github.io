使用Github Action实现全自动部署

Github Action 简介

Github Action是 GitHub 于 2018 年 10 月推出的一个 CI\CD 服务。通过 Github Action，我们只需要在每次完成博客的编写或修改以后，将改动直接 push 到远程仓库，之后的编译部署的工作统统交给 CI 来完成即可。

Github Action 配置教程

获取tocken

为了确保交由 Github Action 来持续部署时，Github Action 具备足够的权限来进行 hexo deploy 操作，需要先获取 Token。

访问 Github-> 头像（右上角）->Settings->Developer Settings->Personal access tokens->generate new token, 创建的 Token 名称随意，但必须勾选 repo 项。选择到期时间永不过期。

创建好了以后记录下Token。

重新设置远程仓库和分支

  1. 先把themes/主题文件夹下的.git 删除掉，原因是主题文件夹下的.git 文件夹的存在会导致其被识别成子项目，从而无法被上传到源码仓库。

  2. 添加屏蔽项
    因为能够使用指令进行安装的内容不包括在需要提交的源码内，所有我们需要将这些内容添加到屏蔽项，表示不上传到 github 上。这样可以显著减少需要提交的文件量和加快提交速度。

  3. 提交博客源码到私有仓库
    在博客根目录下启动终端，使用 git 指令重设仓库地址。这样在新建仓库，我们仍旧可以保留珍贵的 commit history，便于版本回滚。

  4. 查看部署情况
    打开 GIthub 存放源码的私有仓库，找到 Actions。根据刚刚的 Commit 记录找到相应的任务

    若打钩，恭喜你，现在可以开始享受自动部署了。
